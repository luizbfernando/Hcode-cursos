Tome muito cuidado ao desenvolver um arquivo css, há muitos Erros que podem ser cometidos e que comprometem totalmente o uso do arquivo. Veja agora os erros mais comums de se cometer.

Erro de sintaxe

Escrever errado, como faltar ou adicionar letras ao nome do arquivo ou tag fará com que ele não seja carregado. (O css é case sensitive)

Diretório Incorreto

Sempre que for mudar seu arquivo de pasta, certifique-se de mudar também o diretório do link associado a tag 'link'

Seletores e Parentela

Os seletores devem estar escritos corretamente, e a parentela também como por exemplo: .body .main { background color: black; } Este código funcionará somente quando a classe .main estiver dentro da tag com a classe .body sendo a tag .body, a "mãe";
.body > .main = Aplique somente quando a classe .main for descendente direto da classe .body

Especifidade

Quanto mais específico for a declaração, mais importância ela terá. Em caso de não conseguir encontrar o conflito de especifidade, use o infíxo !important (sempre evite usar), exemplo :
.body .main { background-color: black; }
.main { background-color: red!important; }
