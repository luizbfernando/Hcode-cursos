Variáveis nativas

Também conhecidas como built-in, são variáveis que não podem ser alteradas (como o const do javascript).

Escopo de variáveis

As variáveis declaradas no nível superior de uma folha de estilo são globais

Isso significa que elas podem ser acessadas em qualquer lugar em seu módulo após terem sido declaradas.

Mas isso não é verdade para todas as variáveis. Aquelas declaradas em blocos(chaves no SCSS ou código identado no CSS) são geralmente locais e só podem ser acessadas dentro do bloco em que foram declaradas(como o let no javascript)

Shadowing - sombreamento

Variáveis locais podem ser declaradas com o mesmo nome de uma variável global.

Se isso acontecer, na verdade existem duas variáveis diferentes com o mesmo nome: uma local e outra global.

Isso ajuda a garantir que um autor escrevendo uma variável local não altere acidentalmente o valor de uma global da qual ele sequer tem conhecimento.

Utilizando variáveis built-in

@use "sass:math" as math;
math.$pi: 0;

Arquivo style.scss

$variavel-global: 50px;

.content {
$variavel-local: 20px;
margin-top: $variavel-global;
margin-left: $variavel-local;
}
.sidebar {
margin-top: $variavel-global;
}

Erro: margin-left: $variavel-local;

Arquivo style.css

.content {
margin-top: 50px;
margin-left: 20px;
}

.sidebar {
margin-top: 50px;
}
